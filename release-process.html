Release process

更新日: 2018.06.14

注意：このページはコアコミッターを対象としています。

  • CHANGES.mdを、最終更新以降の関心の高い内容で更新して下さい。
  • 3箇所のバージョン(x.y.z表記)を更新して下さい。
    • CHANGES.mdのヘッダー(これはサイトが最新のバージョン番号を探す要素です)。
    • package.jsonのversion属性。
    • docs/conf.pyにある2箇所。(versionとrelease)
  • バージョンの変更をcommitし、バージョン番号でtagを付けてください。 ("v."のような文字は不要です。)
  • commitとtagをpushしてください。 (git push && git push --tags)

tagをpushすると、https://highlightjs.org/api/release/で監視可能な更新プロセスがトリガーされます。

何かが動かず、コードで修正可能な時(バージョン番号の不一致、直前のパッチ等)、 バージョン番号の3桁目(末尾、リビジョン)の番号をインクリメントしてください。
